# Web渗透测试系统实现报告

## 1. 系统实现概述

本项目成功实现了一个面向中小企业的Web渗透测试系统，基于Python 3.12和Django框架，完成了信息收集的核心功能，包括端口扫描、目录爆破和C段探测。系统采用了模块化设计，具有良好的可扩展性和可维护性。

## 2. 技术栈使用情况

| 技术/工具 | 版本 | 用途 |
|---------|------|------|
| Python | 3.12 | 开发语言 |
| Django | 3.2+ | Web框架 |
| SQLite | 内置 | 数据库 |
| Bootstrap | 5.3.0 | 前端框架 |
| dirsearch | 内置 | 目录爆破字典 |
| requests | 最新版 | HTTP请求库 |

## 3. 系统架构设计

### 3.1 目录结构

```
pentest_system/
├── pentest_system/          # 项目配置目录
│   ├── __init__.py
│   ├── settings.py         # 项目配置
│   ├── urls.py             # 主URL配置
│   └── wsgi.py
├── scanning/               # 信息收集模块
│   ├── __init__.py
│   ├── admin.py
│   ├── apps.py
│   ├── models.py           # 数据模型
│   ├── views.py            # 视图函数
│   ├── urls.py             # 模块URL配置
│   ├── utils/              # 工具模块
│   │   └── scanner.py      # 扫描工具实现
│   └── templates/          # 模板目录
│       └── scanning/       # 扫描相关模板
├── manage.py               # 管理脚本
└── db.sqlite3              # SQLite数据库
```

### 3.2 模块划分

1. **核心模块**：
   - scanning：信息收集模块，包含端口扫描、目录爆破、C段探测功能

2. **工具模块**：
   - scanner.py：实现了PortScanner、DirScanner、CSegmentScanner三个核心扫描类

3. **数据模型**：
   - ScanTask：扫描任务模型
   - PortScanResult：端口扫描结果模型
   - DirScanResult：目录扫描结果模型
   - CSegmentResult：C段探测结果模型

4. **视图函数**：
   - scan_task_list：扫描任务列表
   - create_port_scan：创建端口扫描任务
   - create_dir_scan：创建目录爆破任务
   - create_csegment_scan：创建C段探测任务
   - scan_result_detail：查看扫描结果详情

## 4. 功能模块实现详情

### 4.1 端口扫描

**实现方式**：
- 基于Python的socket模块实现TCP端口扫描
- 使用多线程提高扫描速度
- 支持端口范围扫描
- 自动识别常见服务

**核心代码**：
- `PortScanner`类：实现端口扫描逻辑
- `scan_port`方法：扫描单个端口
- `get_service`方法：根据端口号识别服务

**使用方式**：
- 填写任务名称、目标IP/域名和端口范围
- 系统会自动执行扫描并保存结果

### 4.2 目录爆破

**实现方式**：
- 集成dirsearch的字典文件
- 使用Python的requests库发送HTTP请求
- 支持自定义文件扩展名
- 多线程并发扫描

**核心代码**：
- `DirScanner`类：实现目录爆破逻辑
- `get_default_wordlist`方法：获取默认字典
- `scan_path`方法：扫描单个路径

**使用方式**：
- 填写任务名称、目标URL和可选参数
- 系统会使用内置字典进行扫描

### 4.3 C段探测

**实现方式**：
- 基于Python的socket模块实现ICMP和TCP探测
- 多线程并发探测
- 自动扫描常见端口

**核心代码**：
- `CSegmentScanner`类：实现C段探测逻辑
- `scan_ip`方法：探测单个IP地址
- `scan_ports`方法：扫描IP的常见端口

**使用方式**：
- 填写任务名称和目标IP
- 系统会自动探测同C段的存活主机

## 6. 系统特点

1. **模块化设计**：系统采用模块化设计，便于扩展和维护
2. **多线程扫描**：使用多线程提高扫描速度
3. **集成开源工具**：集成了dirsearch的字典文件，提高目录爆破效果
4. **友好的用户界面**：使用Bootstrap框架，提供直观的Web界面
5. **完整的结果展示**：详细展示扫描结果，便于分析
6. **安全的权限控制**：基于Django的认证系统，确保只有授权用户可执行扫描
