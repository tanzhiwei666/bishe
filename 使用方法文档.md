# Web渗透测试系统使用方法文档

## 系统概述

本系统是一个面向中小企业的Web渗透测试系统，基于Django框架开发，实现了信息收集的核心功能，包括端口扫描、目录爆破和C段探测。系统使用Python 3.12和Django 3.2+，集成了开源工具dirsearch用于目录爆破。

## 系统环境要求

- Python 3.12
- Django 3.2+
- PostgreSQL（可选，默认使用SQLite）
- 依赖库：requests

## 系统安装与配置

### 1. 安装依赖

```bash
# 进入项目目录
cd pentest_system

# 安装依赖
pip install -r requirements.txt
```

### 2. 数据库迁移

```bash
# 生成迁移文件
python manage.py makemigrations

# 执行迁移
python manage.py migrate
```

### 3. 创建超级用户

```bash
# 创建超级用户
python manage.py createsuperuser

# 或者使用命令行直接创建（用户名：admin，密码：admin123）
python manage.py createsuperuser --username admin --email admin@example.com --noinput
python manage.py shell -c "from django.contrib.auth.models import User; u=User.objects.get(username='admin'); u.set_password('admin123'); u.save()"
```

### 4. 启动开发服务器

```bash
# 启动开发服务器
python manage.py runserver

# 访问地址
# http://localhost:8000/scanning/
```

## 功能使用方法

### 1. 登录系统

- 访问 `http://localhost:8000/admin/` 登录管理后台
- 使用超级用户账号登录（用户名：admin，密码：admin123）
- 或者直接访问 `http://localhost:8000/scanning/`，系统会自动跳转到登录页面

### 2. 端口扫描

1. 在扫描任务列表页面，点击"端口扫描"按钮
2. 填写任务名称、目标IP/域名和端口范围
3. 点击"开始扫描"按钮
4. 扫描完成后，点击"查看结果"查看扫描结果

**参数说明**：
- 任务名称：自定义任务名称，便于识别
- 目标IP/域名：要扫描的目标IP地址或域名
- 端口范围：可以是单个端口（如80）或端口范围（如1-1000）

### 3. 目录爆破

1. 在扫描任务列表页面，点击"目录爆破"按钮
2. 填写任务名称、目标URL和可选参数
3. 点击"开始扫描"按钮
4. 扫描完成后，点击"查看结果"查看扫描结果

**参数说明**：
- 任务名称：自定义任务名称，便于识别
- 目标URL：要扫描的目标网站URL（如http://example.com）
- 字典路径：可选，默认使用内置字典（dirsearch的common.txt）
- 文件扩展名：可选，多个扩展名用逗号分隔（如php,html,js）

### 4. C段探测

1. 在扫描任务列表页面，点击"C段探测"按钮
2. 填写任务名称和目标IP
3. 点击"开始探测"按钮
4. 扫描完成后，点击"查看结果"查看探测结果

**参数说明**：
- 任务名称：自定义任务名称，便于识别
- 目标IP：要探测的目标IP地址（如192.168.1.1）

## 结果查看

1. 在扫描任务列表页面，找到已完成的任务
2. 点击"查看结果"按钮
3. 系统会根据扫描类型显示不同的结果表格：
   - 端口扫描：显示端口、状态、服务、版本
   - 目录爆破：显示URL、状态码、大小、内容类型
   - C段探测：显示IP地址、状态、开放端口

## 注意事项

1. **合法使用**：本系统仅用于合法的安全测试，请勿用于未经授权的测试
2. **扫描强度**：扫描可能会对目标系统造成一定的负载，请合理设置扫描参数
3. **网络环境**：C段探测和端口扫描可能会触发网络安全设备的告警
4. **性能考虑**：扫描大量端口或目录可能会消耗较多系统资源

## 常见问题

### 1. 扫描任务执行失败

**可能原因**：
- 目标不可达
- 网络连接超时
- 系统资源不足

**解决方案**：
- 检查目标是否可访问
- 调整超时设置
- 减少扫描范围

### 2. 目录爆破没有结果

**可能原因**：
- 目标网站有WAF防护
- 字典太小，没有匹配的路径
- 网络连接问题

**解决方案**：
- 使用更大的字典
- 调整请求头和延迟设置
- 检查网络连接

### 3. C段探测没有发现主机

**可能原因**：
- 目标网络没有其他主机
- 网络访问受限
- 探测方式不适合目标网络

**解决方案**：
- 确认目标网络是否有其他主机
- 检查网络访问权限
- 尝试不同的探测方式
